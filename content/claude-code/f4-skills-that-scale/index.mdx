# F4: Skills that scale past solo usage

Most skills fail at team scale because they are written for one person's assumptions.

## What we tried

We kept only skills that were:

- role-agnostic
- input-explicit
- output-structured
- testable in review

## What happened

Reuse went up and onboarding time dropped because teammates could run the same skill and get comparable results.

## What we learned

- Skills should encode team standards, not personal style.
- Structured outputs make downstream review faster.
- Prune aggressively: one high-quality skill is better than five vague ones.
