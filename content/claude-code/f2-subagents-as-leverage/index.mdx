# F2: 5x output without 5x tokens: subagents as leverage

Subagents improved throughput only after we redesigned task boundaries.

<WarStory title="Parallelism without ownership created rework">
We had three subagents working on a feature simultaneously. One wrote to a shared utility file, another rewrote the same function differently, and the third assumed a function signature that hadn't been finalized. The merge took longer than building the feature from scratch would have. The problem wasn't parallelism — it was that we'd assigned overlapping ownership.
</WarStory>

## What we tried

We split work into parallel, non-overlapping units with one integration owner.

## What happened

Cycle time dropped and review quality improved because each delegated output had a clear acceptance shape.

## What we learned

- Delegate bounded work, not ambiguous work.
- Assign one owner for final integration.
- Measure leverage by lead time and rework rate, not output volume.

## Result

Lead time on feature branches dropped from an average of three days to under one day for bounded tasks. The change wasn't purely about subagents — it was about task design. We now scope every task before delegating: clear input, clear output shape, clear file boundary. That habit changed how we wrote tickets too. The rework rate went from roughly one in three delegated tasks requiring major revision to about one in eight.
