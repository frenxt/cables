# Day 9: Your first subagent

Subagents are most useful when the task is bounded and parallelizable.

## What we tried

We created a narrow "reviewer" subagent to inspect one module at a time while the primary agent handled integration.

## What happened

When we gave clear ownership boundaries, delegated work became faster and easier to merge. Broad prompts produced noisy results.

## What we learned

- Delegate only well-scoped tasks.
- Assign explicit file ownership for each subagent.
- Keep the integration path in the primary agent.
