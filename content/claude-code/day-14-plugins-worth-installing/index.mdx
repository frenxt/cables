# Day 14: Plugins worth installing

Most plugin bloat comes from installing tools before defining workflows.

## What we tried

We limited installs to four plugin classes:

- repo and PR operations
- issue tracker integration
- observability lookup
- deployment status checks

## What happened

Command palette noise dropped and usage rates went up because each plugin mapped to a real recurring task.

## What we learned

- Install only plugins tied to weekly workflows.
- Remove anything unused after two sprints.
- Document plugin purpose in your team CLAUDE.md.
