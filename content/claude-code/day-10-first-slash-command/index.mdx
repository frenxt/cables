# Day 10: Your first slash command

Slash commands are the fastest way to package a routine you run every week.

## What we tried

We created a release-readiness slash command that always checks the same five release gates.

## What happened

It reduced checklist drift and made release prep predictable.

## What we learned

- Use slash commands for repeatable routines, not deep reasoning.
- Keep command output structured and checklist-shaped.
- Version the command when process requirements change.
