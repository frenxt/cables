# Day 3: What lives in `~/.claude/` and why

Most teams discover `~/.claude/` accidentally, usually after installing a skill from someone else. We ignored it early on and kept wondering why some commands appeared in one project but not another.

## What we tried

We mapped everything in `~/.claude/` and compared it with project-local `.claude/` folders:

- `~/.claude/skills/` for reusable skills across projects
- `~/.claude/commands/` for slash commands
- `~/.claude/agents/` for subagent prompts

Then we tested the same command in two repos to see what resolved globally vs locally.

## What happened

Global assets worked exactly where we expected, but only once naming was consistent. Local `.claude/` content is where project-specific behavior should live; global is for patterns you want everywhere.

## What we learned

- Put opinionated project behavior in the repo-local `.claude/` directory.
- Put reusable utility skills in `~/.claude/skills/`.
- Prefix names by purpose (`review-*`, `debug-*`, `release-*`) so the command palette stays searchable.
- If behavior feels "random," check whether you installed an item globally or locally.
