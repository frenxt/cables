# Day 13: CLAUDE.md patterns: when to split

A single `CLAUDE.md` works until project complexity exceeds one file's clarity budget.

## What we tried

We kept a root `CLAUDE.md` for universal rules and moved domain-specific instructions into folder-local docs referenced from the root.

## What happened

Context quality improved because each area had tighter, relevant guidance instead of one oversized instruction document.

## What we learned

- Keep global rules global; keep domain rules close to domain code.
- Split when edits become frequent and unrelated sections start colliding.
- Add a lightweight table of contents so navigation stays fast.
