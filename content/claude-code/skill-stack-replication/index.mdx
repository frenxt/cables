# Replicate Ragav's stack (skills + plugins + scripts)

This cable installs the actual stack artifacts from Ragav's `~/.claude*` and `~/.codex` setup so anyone can replicate it quickly.

<WarStory title="Same prompt, different output across profiles">
The gap was stack drift: different skills and plugins loaded in different profiles. This pack removes that drift by shipping one installable baseline.
</WarStory>

## One command to install the pack

```bash
npx frenxt-cables add skill-stack-replication
```

What gets installed in your project:

- Core personal skills (`analyse-langsmith-trace`, `benchmark`, `e2e-review`, `qa`, `bugfix-cycle`, `debug-prompt-caching`, `qa-release`, `voice-test`)
- Claude + Codex plugin manifests and install script
- Bootstrap scripts
- Category manifests for selective copy/install

## Subcategory replication (your taxonomy)

- **Developer Workflow Systems**
- **Reasoning Frameworks**
- **Domain Playbooks**

After install, copy by subcategory:

Install only category plugins (Claude + Codex):

```bash
bash .claude/stacks/frenxt/install-category-plugins.sh developer-workflow-systems
```

Sync only category skills to `~/.claude*`:

```bash
bash .claude/stacks/frenxt/sync-category-skills-to-profiles.sh developer-workflow-systems
```

Accepted category values:

- `developer-workflow-systems`
- `reasoning-frameworks`
- `domain-playbooks`

## Full replication

```bash
bash .claude/stacks/frenxt/bootstrap-frenxt-stack.sh
```

## Why this works

- Shared baseline skills remove behavior drift.
- Category manifests let readers adopt only the parts they need.
- Plugin setup becomes repeatable across both tools with explicit manifests instead of manual memory.
